单例模式
简单来说单例模式就是一个类只能创建一个对象，以下是需要注意的几点
1，私有化构造函数
2，防止多线程创建多个对象

优点
1，减少内存开支，特别是频繁创建、销毁时，而且创建销毁时性能无法优化
2，避免对资源的多重占用
3，可以在系统设计全局的访问点，优化和共享资源访问

缺点
1，单例一般没有接口扩展很困难，单例一般没有接口
2，单例对测试是不利的，没有接口不能使用mock的方式虚拟一个对象
3，单例与单一原则有冲突，一个类应该只实现一个逻辑，而不关系它是否是单例

使用场景
1，要求生成唯一序列号的环境
2，整个项目中需要一个共享访问点或共享数据
3，创建一个对象需要消耗的资源过多
4，定义大量的静态常量和静态方法的环境

命令模式
高内聚模式
将一个请求封装成一个对象，从而让你使用不同的请求把客户端参数化，对请求排队或者记录请求日志，可以提供命令的撤销和恢复功能

优点
1，类间解耦 调用者和接收者角色之间没有任何依赖关系，调用者实现功能时只需要调用Command抽象类的execute方法就可以了，不需要了解底层是哪个接收者执行
2，可扩展性 Command类可以非常容易扩展，而调用者Invoker和高层次的模块Client不产生严重的代码耦合

与其他模式结合
1，可以和责任链模式，实现命令解析任务
2，结合模板方法模式，则可以减少Command子类膨胀问题

缺点
1，Command的子类是有多少命令就有多少个，所以可能会出现类膨胀

适用场景
1，认为是命令的地方都可以采用命令模式，例如一个按妞的点击命令

反悔功能
1，结合备忘录模式还原最后状态
